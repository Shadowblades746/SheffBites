Design

Introduction

  • When initially designing my website, I decided to research colour schemes.
  • My aim was to create a website that looked inviting and friendly while being practical and this meant that my colour scheme had to be bright while also being professional.
  • I wanted to ensure that the reader could efficiently take in the information of my website in an engaging and appealing way which determined the way I laid out each of my elements.

Site map

Design mock ups

  • I used the website Colour Scheme and decided to go with "wheat" as my main background with "teal" and "coral" as my header and footer background respectively.
  • By using complementary colours, there was high contrast, in combination with a flowy design which helps keep the reader engaged in the website.
  • I ensured every image was rounded and had a shadow to make it pop, and each footer menu conformed to the same style to keep consistency.
  • I used "hr" elements to separate sections of each webpage to make it easier to read and distinguish between separate areas.
  • I used flexbox to make images stack vertically on mobile and horizontally on desktop.
  • I styled each link in bold and added a hover effect to ensure they were clear while not being jarring.
  • I appropriately used text decoration, font weighting and font size to highlight each element.
  • I appropriately used element backgrounds for certain elements to differentiate them from the background and encapsulate the child elements.
  • I made use of semantic elements to style my website appropriately.
  • I made use of transitions and effects such colour change and element resizing to make my website more engaging.
  • I made use of a CSS normalize template to ensure that my designs looked the same no matter what web browser was used.
  • I made use of scaling elements such as "rem" to ensure my website scaled correctly.

Breakpoint

  • I made use of a single breakpoint at 600px as it was much larger than the average mobile display size as shown by Mobile display sizes so that no mobile would ever have a desktop display.
  • This kept my website mobile first since my breakpoint used a minimum pixel size rather than maximum.

Menu system

  • I used my own menu system.
  • I added a header bar with a logo in the left and links in text form aligned to the right.
  • I added a footer bar with a copyright statement in the right and links in text form aligned to the left.
  • On a mobile display. the header links become a hamburger menu which displays a vertical list of links when tapped on, and when the user taps outside the hit-box, the links collapse into the hamburger.
  • On a mobile display, the horizontal footer morphs into a vertical footer with vertical links and the copyright statement at the bottom.

Browser testing

  • I have tested my website on both a chromium and quantum based browser using both a desktop and mobile view, to ensure that it runs perfectly no matter how the user views it.
  • I have not shown every single page as it would be redundant, but as seen in the screenshots, both browser engines display the website as intended with no side effects.

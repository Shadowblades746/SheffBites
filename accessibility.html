Accessibility Statement

Measures to Support Accessibility

I tested my website using tools such as

  • Web Content Accessibility Guidelines (WCAG) 2.2
  • Narrator | Microsoft Windows
  • WebAIM: WebAIM's WCAG 2 Checklist
  • WAVE Web Accessibility Evaluation Tools
  • The W3C Markup Validation Service
  • The W3C CSS Validation Service

I have included features such as

  • alternative text: ensures that if images do not load then there is descriptive text that describes each image.
  • figures: semantic elements that ensure screen readers know where each figure is on the page.
  • figure captions: help screen readers and anyone to check what an image is with embedded links to the source image.
  • form labels: help screen readers correctly read the form.
  • language: help screen readers correctly identify the language of the page to read content appropriately.

I have included structural semantic elements which help with navigation for screen readers such as

  • headings
  • unordered lists
  • navigation
  • header
  • main
  • footer

I have included ARIA elements which help with screen readers when HTML semantic elements are not descriptive enough such as:

  • ARIA labels

Status of Compliance

My website is partially compliant with WCAG 2.1 Level AA and fully compliant with CSS level 3 + SVG.

Assistive technologies supported

I have tested my website with screen readers such as the Windows narrator, as well as keyboard-only navigation, to ensure accessibility for all users.
